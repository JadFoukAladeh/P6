Purpose

The purpose of this HTML page is to be used as the test plan for Project VI. The will guide the users through the testing needed for Project VI. This test plan will fall into 5 categories as seen below:

  1. Environmental Setup - Ensuring all users have the same initial testing setup
    • Software Download & Installation
    • Initial Virtual Machine Setup
    • Initial Server Setup
    • Cloning Github Code
  2. Raspberry PI (RPi) Server Launching
  3. CAN Bus Testing
  4. Communications Between RPi, Arduino, and STM32F303RE
  5. Final Testing
ASSUMPTIONS:
1. Users are familiar with programming STM32 boards using Keil uVision5 and have it installed on testing system
2. Users are familiar with using linux and using the linux terminal
3. Users are familiar with programming STM32 boards using STM32CubeIDE 1.15.1 and have it installed on testing system
4. Users are familiar with C programming
5. Users are familiar with programiming Arduinos and have Arduino IDE installed on testing system

Environmental Setup

Software Download & Installation

For the purpose of testing this project the users are expected to be running Windows 10. This is to ensure the same testing environment on all computers to minimize errors in testing and findings. This section of the test plan will guide the users through the environment setup procedure that is required. Following this procedure will allow the user to be able to:

  • Launch a server from the Raspberry Pi
  • Connect remotely to a Raspberry Pi through local network and through VPN using VNC and SSH
  • Communicate with the server from a virtual machine
  • Use github to obtain the current code for the repository to be used for testing
There are 5 main software packages that need to be downloaded, which are as follows:
  1. Ubuntu 24.04 LTS - Image of Ubuntu being used
  2. Oracle VM VirtualBox
  3. VNC
  4. PuTTY
  5. Ivanti Secure Access Client

Ubuntu 24.04 LTS

  1. Open the following link: Ubuntu
  2. Download the latest Ubuntu package for Windows hosts - current version (24.04 LTS)

Oracle VM VirtualBox

  1. Open the following link: Oracle VM VirtualBox
  2. Download the latest VirtualBox platform packages for Windows hosts - current version (7.0.18)

VNC

  1. Open the following link: VNC
  2. Download the latest VNC packages for Windows hosts - current version (7.11.1)

PuTTY

  1. Open the following link: PuTTY

Ivanti Secure Access Client

  1. Open the following link: Ivanti Secure Access Client

  • Install the Oracle VirtualBox from the executable file downloaded above
  • Install the PuTTY from the executable file downloaded above
  • Install the Ivanti Secure Access Client from the executable file downloaded above

    • Initial Virtual Machine Setup

    1. Launch Oracle VirtualBox
    2. Press the "New" button to create a new virtual machine(VM)
    3. Name the VM something meaningful such as Project VI testing in the Name field
    4. In the ISO Image field press the drop down menu. Click Other and scroll to the Downloads folder and choose the ubuntu iso we downloaded in the previous step.
    5. Check the box to Skip Unattended Installation and click Next.
    6. Open the settings tab (while VM is off) and press Network
    7. There will be a drop down menu next to the phrase "Attachted to:". In the drop down menu, ensure Bridged Adapter is chosen. This step will ensure that our VM has a unique IP address assigned through DHCP so that we can access our website over the web.
    8. Set a Base Memory of 8048MB and click next.
    9. Set a Disk Size of 100 GB and click next. Then click Finish.
    10. Launch the VM and go through initial setup. The remaining commands will all be executed in the VM
    11. The following commands will download and install LAMPP server which will be used to host an HTML server. This installation method will run the HTML server automatically upon the launch of the OS. Open the terminal and type the following commands:
      • sudo apt-get install net-tools
      • sudo apt-get install lamp-server^
      • sudo apt-get install git

    Initial Server Setup

    1. sudo apt-get install code - this will install Visual Studio Code
    2. Open Visual Studio Code install the following extensions: click Ctrl+Shift+x to open the Extensions tab. Extensions can be installed by clicking the install button after searching for the extension needed.
      • GitLens - Git supercharged
      • Auto Close Tag
      • Babel JavaScript
      • Color Highlight
      • file-icons
      • GitHub Pull Requests
      • PHP Intelephense
      • Prettier - Code formatter
      • Remote - SSH
      • Remote - SSH: Editing Configuration Files
      • Remote Explorer
      • vscode-icons
    3. Within the terminal input the following cd /var/www/html
    4. touch index.html | sudo chmod -R 777 index.html - this will create a file which will be the index to the server linking all pages of the website. This file will be initialized with read, write, and execute priviliages.
    5. In the explorer tab in Visual Studio Code on the left hand side of the screen, you should now see index.html
    6. Open the index.html by clicking on it in Visual Studio Code. Type ! followed by Enter. This will create a dummy server that can be seen through a broweser
    7. In the terminal type hostname -I - this will give you the IPV4 address of the computer. Type this in the search bar of google chrome or firefox. This should launch a blank html page. Which means your server is now setup.

    Cloning github code

    • By this point the testing environment is setup. This step will clone all the up-to-date code from the github repository allowing the users acess to the code that can be used in the test envioronment.
    • Within the terminal input the following commands in order:
      1. cd /var/www/html
      2. git clone https://github.com/JadFoukAladeh/P6.git
    • All the code has now been cloned into a new repository P6. Traverse to P6 using the following command in the terminal: cd P6/

    Communications Between RPI, Arduino, and STM32F303RE

    The purpose of this step is to ensure CAN communiccation between the 6 nodes (4 STM boards, 1 Arduino board, 1 RPi). Each node will be expected to recieve and transmit to the other nodes.The code for the STM boards will be found in this repository in a file called code. The code will be labeled as comm_demo_code. The code for the Arduino will be labeled comm_code. This code will enable us to show the current sender ID on the LCD attached to the elevator. The RPi should have the front_end and back_end at this point to enable communication through the HTML page to the PHP code.

    1. Copy the code and upload it through STM32CubeIDE 1.15.1 to all STM boards attached to elevator
    2. Copy the code and upload it through Arduino IDE to the Arduino board attached to elevator
    3. Press the blue button on one of the STM boards. LED2 then will turn on for 2 seconds on one board at a time. I It will cycle through each STM board. Once that is complete, the RPi will send a message back and should light each LED2 on all the STM boards.
    4. The user simultaneously should see a message on the LCD display attached to the elevator. This message should have the unique ID of the board transmitting the message
    5. For this comms demo the STM boards will have the filters set to accept the RPi's ID and Arduino's ID. This will ensure that when a message is sent from the RPi or Arduino, each STM board on the CAN bus when listening for data can pick up on it.
    6. For this comms demo the Arduino board will have the filters set to accept the RPi's ID and STM boards IDs. This will ensure that when a message is sent from the RPi or STM boards, the Arduino board on the CAN bus when listening for data can pick up on it.
    7. For this comms demo the RPi boards will have the filters set to accept the Arduino's ID and STM boards IDs. This will ensure that when a message is sent from the Arduino or STM boards, the RPi board on the CAN bus when listening for data can pick up on it.

    Final Testing

    The purpose of this step is to final test the elevator with all components working to spec. Each node will be expected to recieve and transmit to the other nodes.The code for the STM boards will be found in this repository in a file called code. The code will be labeled as final_code. The code for the Arduino will be labeled final_code. This code will enable us to see the current floor the elevator is on and what state the door is currently in whether open or closed on the LCD attached to the elevator. The RPi should have the front_end and back_end at this point to enable communication through the HTML page to the PHP code. This demo is meant to be presented in person using a laptop that will connect over the elevator network.

  • Copy the code and upload it through STM32CubeIDE 1.15.1 to all STM boards attached to elevator
  • Copy the code and upload it through Arduino IDE to the Arduino board attached to elevator

    • VNC or SSH into the RPi and open the terminal. In the terminal type the following 2 commands one at a time to clone the latest code to update the server:
    cd /var/www/html
    git clone https://github.com/JadFoukAladeh/P6.git

  • This demo will be demonstrated in two methods:
    1. Method 1: Using the HTML page

      3. This should open up an HTML page that allows the elevator to be controlled through the broweser using a GUI.
      4. The GUI will have 5 buttons: floor 1, floor 2, floor 3, open door, close door
      5. User can now input a button press through the GUI and watch the elevator react. Whether go to a certain floor or open and close the door
    2. Method 2: Using the physical buttons
      1. User can now input a button press through on the physical buttons and watch the elevator react. Whether go to a certain floor or open and close the door
